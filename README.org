#+title: nucleo-completion

~nucleo-completion~ is an Emacs completion style backed by a Rust dynamic
module using the ~nucleo-matcher~ crate.

It also works without the Rust module by falling back to a simple Emacs Lisp
subsequence matcher.

* Setup

Install from MELPA:

#+begin_src text
  M-x package-install RET nucleo-completion RET
#+end_src

Or install directly from the repository with ~package-vc~:

#+begin_src emacs-lisp
  (package-vc-install
   '(nucleo-completion
     :url "https://github.com/kn66/nucleo-completion.el"))
#+end_src

Then enable the completion style:

#+begin_src emacs-lisp
  (require 'nucleo-completion)
  (add-to-list 'completion-styles 'nucleo)
#+end_src

Or configure it with ~use-package~:

#+begin_src emacs-lisp
  (use-package nucleo-completion
    :vc (:url "https://github.com/kn66/nucleo-completion.el.git"
         :rev :newest)
    :config
    ;; Optional: install and load the Rust module when it is missing.
    (nucleo-completion-ensure-module)

    (setopt completion-styles '(nucleo basic))
    (add-to-list 'completion-category-overrides
                 '(eglot-capf (styles nucleo basic)))

    (setopt nucleo-completion-sort-ties-by-length t
            nucleo-completion-sort-ties-alphabetically t
            nucleo-completion-sort-ties-by-history t
            nucleo-completion-highlight-score-bands t))
#+end_src

The package works without a Rust dynamic module by using a simple Emacs Lisp
fallback matcher.  The fallback filters candidates but does not apply
Rust-backed scoring or tie sorting.

To use a local checkout, add the package directory to ~load-path~:

#+begin_src emacs-lisp
  (add-to-list 'load-path "/path/to/nucleo-completion.el")
  (require 'nucleo-completion)
  (add-to-list 'completion-styles 'nucleo)
#+end_src

* Rust Dynamic Module

The Rust module is optional.  It enables Nucleo scoring, tie sorting, and
score-aware highlighting; without it, candidates are filtered by the Emacs Lisp
fallback.

The MELPA package does not include prebuilt binary modules.  To install a
module for the current platform from GitHub Releases, run:

#+begin_src text
  M-x nucleo-completion-install-module
#+end_src

The command asks before downloading, verifies the published SHA256 checksum,
stores the file under ~nucleo-completion-module-directory~, and tries to load
it.  No download happens during package installation or ordinary package load.
If a module was already loaded in the current Emacs process, restart Emacs to
use the newly installed file.

To opt in to automatic install from your configuration, call:

#+begin_src emacs-lisp
  (nucleo-completion-ensure-module)
#+end_src

This first tries to load an already installed module.  If no module is
available, it downloads and installs the prebuilt module for the current
platform.  Lisp calls install without asking;
~M-x nucleo-completion-ensure-module~ asks before downloading.

By default, the installer uses GitHub's latest release URL.  To pin a release,
set ~nucleo-completion-module-release-tag~ before installing:

#+begin_src emacs-lisp
  (setq nucleo-completion-module-release-tag "v0.1.11")
#+end_src

Check whether the Rust module loaded with:

#+begin_src emacs-lisp
  (featurep 'nucleo-completion-module)
#+end_src

If a readable module is found but cannot be loaded, the failure details are
saved in ~nucleo-completion-module-load-errors~.  Set
~nucleo-completion-report-module-load-errors~ before loading the package to also
show those failures in ~*Messages*~.

To ask once at interactive startup when no module is available, set:

#+begin_src emacs-lisp
  (setq nucleo-completion-module-install-policy 'prompt)
#+end_src

* Local Build

#+begin_src sh
  cargo build --release
#+end_src

The loader also checks ~target/release~ and ~target/debug~, so no extra
~load-path~ entry is required when building inside the package directory.

Emacs must be built with dynamic module support. You can check this with
~(functionp 'module-load)~.

* Prebuilt Release Assets

The GitHub Actions workflow in ~.github/workflows/prebuilt-modules.yml~ builds
release artifacts named
~nucleo-completion-module-<target-triple>.<extension>~ plus matching
~.sha256~ files for:

- ~x86_64-unknown-linux-gnu~
- ~x86_64-apple-darwin~
- ~aarch64-apple-darwin~
- ~x86_64-pc-windows-msvc~

Run the workflow manually with the release tag to upload assets to an existing
GitHub Release.  Linux binaries are sensitive to libc compatibility; for broad
Linux distribution, consider adding a musl release asset.

* Performance

The Rust module scores large candidate sets in worker threads using dynamic
batches.  Emacs Lisp filtering is wrapped in ~while-no-input~ so newer
keystrokes can interrupt stale completion passes while control is in Lisp; the
module call itself is synchronous.

The module returns scores and match indices together, so highlighted characters
track the same match that produced the score without one module call per
candidate.  Regexp expander results are cached for each completion pass, and the
default minimum expander term length is two characters to avoid broad
one-character patterns.

For an external benchmark comparison, see
[[https://github.com/kn66/emacs-completion-bench][emacs-completion-bench]].
It includes results for ~nucleo~ alongside other Emacs completion styles such
as ~hotfuzz~, ~fussy~, ~orderless~, and ~flex~.

* Configuration

** Tie Sorting

By default, ~nucleo-completion~ keeps Nucleo's score order unchanged.  You can
optionally refine candidates with the same Nucleo score:

#+begin_src emacs-lisp
  ;; In minibuffer completion, put candidates from the current history first
  ;; when scores are equal.
  (setq nucleo-completion-sort-ties-by-history t)

  ;; Put shorter candidates first when scores are equal.
  (setq nucleo-completion-sort-ties-by-length t)

  ;; Then sort alphabetically when scores are still equal.
  (setq nucleo-completion-sort-ties-alphabetically t)
#+end_src

When all options are enabled, candidates are ordered by Nucleo score, then
minibuffer history rank, then length, then alphabetical order.  Different
Nucleo scores are not reordered.  History sorting only affects minibuffer
completion with an active history variable; ordinary buffer completion keeps
the Nucleo order unless the length or alphabetical tie-breakers apply.

Tie sorting requires the Rust module.

** Score Band Highlighting

By default, ~nucleo-completion~ only highlights the matching parts of each
candidate.  You can also highlight candidates by score band:

#+begin_src emacs-lisp
  (setq nucleo-completion-highlight-score-bands t)

  ;; Treat candidates scoring at least 85% of the best score as high-score.
  (setq nucleo-completion-high-score-ratio 0.85)

  ;; Emphasize high-score candidates with bold, underline, both, or neither.
  (setq nucleo-completion-high-score-emphasis '(bold underline))
#+end_src

Candidates that contain every search term as an exact word are treated as
high-score candidates regardless of the ratio.  Other scored candidates use
~nucleo-completion-low-score-face~.  Customize
~nucleo-completion-high-score-face~ and ~nucleo-completion-low-score-face~ to
change the visual style.  The default faces avoid background colors so they do
not conflict with completion UI selection highlights or color themes.  Score
band highlighting requires the Rust module.

** Regexp Expanders

~nucleo-completion-regexp-functions~ can add language-specific matchers.
Each function receives one whitespace-separated search term and returns a
regexp string, a list of regexp strings, or nil.
Regexp expander results are cached for each completion pass, so user
configuration normally does not need its own per-term cache.
The default ~nucleo-completion-regexp-minimum-term-length~ is 2, so expanders do
not run for one-character terms unless you lower that option.

Japanese Migemo example:

#+begin_src emacs-lisp
  (with-eval-after-load 'migemo
    (defun my-nucleo-completion-migemo-regexp (term)
      (when (not (string-empty-p term))
        (let ((regexp (downcase (migemo-get-pattern term))))
          (condition-case nil
              (progn
                (string-match-p regexp "")
                regexp)
            (invalid-regexp nil)))))

    (add-to-list 'nucleo-completion-regexp-functions
                 #'my-nucleo-completion-migemo-regexp))
#+end_src

Chinese pyim example:

#+begin_src emacs-lisp
  (with-eval-after-load 'pyim
    ;; pyim documents its cregexp helpers as opt-in utilities.
    (require 'pyim-cregexp-utils nil t)

    (defun my-nucleo-completion-pyim-regexp (term)
      (when (and (fboundp 'pyim-cregexp-build)
                 (not (string-empty-p term)))
        (let ((regexp (pyim-cregexp-build term)))
          (condition-case nil
              (progn
                (string-match-p regexp "")
                regexp)
            (invalid-regexp nil)))))

    (add-to-list 'nucleo-completion-regexp-functions
                 #'my-nucleo-completion-pyim-regexp))
#+end_src

Latin accent example using Emacs' built-in character folding:

#+begin_src emacs-lisp
  (defun my-nucleo-completion-char-fold-regexp (term)
    (when (and (fboundp 'char-fold-to-regexp)
               (not (string-empty-p term)))
      (let ((regexp (char-fold-to-regexp term)))
        (condition-case nil
            (progn
              (string-match-p regexp "")
              regexp)
          (invalid-regexp nil)))))

  (add-to-list 'nucleo-completion-regexp-functions
               #'my-nucleo-completion-char-fold-regexp)
#+end_src

For other languages or input packages, register a small adapter:

#+begin_src emacs-lisp
  (defun my-nucleo-regexp (term)
    ;; Return nil when TERM should not be handled by this backend.
    (my-package-term-to-regexp term))

  (add-to-list 'nucleo-completion-regexp-functions #'my-nucleo-regexp)
#+end_src

Terms are ANDed together. For each term, the built-in fuzzy subsequence matcher
and configured regexp expanders are ORed together.

When the Rust module is available, candidates that only match a configured
regexp expander are treated as high-priority dictionary matches and are
promoted before Nucleo-scored fuzzy matches.  When score-band highlighting is
enabled, these regexp-only matches use the high-score face instead of being
dimmed as low-score fuzzy matches.

To keep ~nucleo~ scoring in Corfu while disabling Migemo/pyim expansion there,
make ~nucleo-completion-regexp-functions~ buffer-local in Corfu buffers:

#+begin_src emacs-lisp
  (add-hook 'corfu-mode-hook
            (lambda ()
              (setq-local nucleo-completion-regexp-functions nil)))
#+end_src
